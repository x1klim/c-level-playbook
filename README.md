# C-Level Playbook

A Claude Code skill: a short handbook of lessons for building things well in a world that runs on AI. Not a process to follow — a way of thinking, with the reasons attached.

## The idea

Here's the irony the whole thing rests on. A model like Claude is *made of* human knowledge, so you'd expect lessons about how to build well to already be in there. Mostly they aren't — not because the model lacks anything, but because the world that needs them is newer than the data that taught it. How work gets done changed when AI arrived: the timeframes, the division of labor, what a "task" even is. The intelligence doing that work was shaped by the world *before* the shift.

So at one of these new forks, its defaults pull it toward the old, pre-AI way — confidently, and for good historical reasons. These lessons fill the gap, written why-first the way a mentor would tell them; it would be a little hypocritical to demand reasoning from a model and then hand it rules with the reasons cut off.

## What's inside

Five short pieces, a few minutes end to end:

- **You don't run on human time** — "for now" and "v2 later" are human reflexes; you have no clock.
- **Tell facts from opinions, for your own sake** — a marked guess is a flag in your *own* reasoning.
- **Foundation up, not facade down** — you'll always reach *a* result; the job is the model under it.
- **Stay above the work** — don't drown in the detail.
- **Which way will you build it?** — two roads, both arrive; the choice is yours.

## Install

```bash
git clone https://github.com/x1klim/c-level-playbook.git ~/.claude/skills/c-level-playbook
```

Claude Code picks it up automatically, or invoke it with `/c-level-playbook`.
